# 回答（問題 10.4）

## date-fns
date-fns は `src/` 直下に多数の関数ファイルを置く細粒度のフラット構成である。各機能は独立モジュールとしてエクスポートされ、必要な関数だけをインポートできるようになっている。`locale/` も言語ごとに独立し、不要なロケールを省ける設計である。  
また、v4 系ではタイムゾーン処理を別パッケージに分離する方針が取られている。モジュール分割の目的は、関数をできるだけ小さく分けてツリーシェイキングを容易にする点にある。

---

## Luxon
Luxon の `src/` は `DateTime`、`Duration`、`Interval`、`Zone` など概念単位でまとめられている。日付や時間に関する主要な型ごとに API が整理され、個々の関数というよりクラスやオブジェクトを中心に利用する構成である。  
このため Luxon は統合的な API を提供することを重視しており、date-fns のような関数粒度の分割とは性質が異なる。モジュール分割の特徴は、強い概念モデルに基づく中粒度のまとまりである。

---

## Day.js
Day.js はコアを極小化し、追加機能を `plugin/` 以下に切り出す構成である。必要に応じて `dayjs.extend` でプラグインを読み込み、ロケールも `locale/` ごとに独立モジュールとして選択的に利用する。  
この方式により本体サイズを抑えつつ、用途に合わせた拡張が可能となっている。モジュール分割の特徴は、極小コアとプラグイン・ロケールの拡張型設計である。
